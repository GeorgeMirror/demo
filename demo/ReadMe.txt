Серверная часть тестового REST приложения.
Работает в двух режимах List и H2
Доступно переключение режимов при работе

Для H2 базы данных не использовался CRUD репозиторий

Тестами покрыты все методы, исполняющие логику.
Для внедренной возможности переключать обработку с List на H2 и назад сохранением данных, репозитории объявлены Singltone.
Это ведет к ошибкам создания строки и получения пустого массива из базы. Впрочем, по отдельности, они проходят.

Команды get запроса: 
/ - покажет весь массив данных из базы данных, отсортировав по дате
/{id} – покажет строку c id = {id}
/produce – создаст 10 элементов в базу данных (повторный вызов не приведет к дополнительному созданию)
/produce_{i} – создаст i элементов в интервале от 1 до 500
/start – создаст 10 элементов, один изменит, один удалит и вернет все данные из базы, отсортировав по дате

Функции create, update ждут на вход post запрос c данными для создания Task
Функция delete ждет post запрос с id
